# プロトタイプ評価アンケート 結果まとめ ＋ 考察・展望

対象：大学生25名（頑張りすぎた経験「あった」23名）。
アプリ評価の自由記述・選択式は回答者により未回答があるため、設問ごとに有効回答数は前後する。
以下、回答は R1〜R25 の匿名番号で示す。

---

## 1. アプリの評価（選択式の集計）

### 日常の中で使ってみたいか（n=22）
- 使ってみたい／どちらかといえば使ってみたい … **14名**
- 使わない／どちらかといえば使わない … **8名**
→ 日常利用の意向は肯定がやや優勢。

### 頑張りすぎている時期に使ってみたいか（n=22）
- 使ってみたい系 … **11名**／使わない系 … **11名**（拮抗）
→ 「疲れている・追われている時こそ使いにくそう」という懸念（後述の改善点）が拮抗の一因。

### アプリ説明としてどちらが「使ってみたい」か（n=22）
- **B：壊れずに成果を出し続けるためのコンディション管理ツール … 14名**
- A：疲れた心を休めるためのセルフケアツール … 6名
- どちらでもない … 2名
→ 全体でも B が優勢。特に**支援対象＝「回復できていない層」15名**では **B 11 : A 3** とさらに明確に B を支持。
　「休める」より「壊れず成果を出し続ける」という前向きな枠組みの方が刺さる。

## 2. アプリの良かった点・気に入った点（好評）

**ビジュアル・雰囲気（最多）**
- アイコン／絵文字／UIが可愛い・癒される・優しい雰囲気（R1・R3・R4・R8・R18・R20・R21・R25）
- 絵文字がシャカシャカ動くインターフェースがユニークで好み（R18）

**音（SE）**
- 音が好き・SEが可愛い。音ありはわくわく、音無しも選べるのがありがたい（R7・R8・R20）

**可視化による客観視・気づき（コンセプトの核が伝わっている）**
- 今日何を・どれくらいしたかが明確にわかる／何に時間を割いているか客観視できる（R6・R16）
- 「思ったより無理していたかも」と自覚につながる（R17・R9・R25・R4）
- 対人関係のイベントを“休息”に数えていない点が良い（R6）
- 回復すると溜まった絵文字が消えていくのが快感（R17）

**仕組み・操作**
- 時間割り振りバーを直接操作できて便利／疲れの強度を設定できる／心と体の疲れやすさを調整できる（R14）
- 通勤・授業などの項目が最初から用意されていて、いちいち登録せず使いやすい（R20）
- 回復できる行為を予定に追加すると回復がたまる仕組みが新しく面白い／細かくカスタマイズできる（R23）

**コンセプト・導入**
- 「シャカシャカできる程度にがんばろう」が“疲れ・キャパ”の感覚とわかりやすく結びつく（R14）
- チュートリアルが丁寧でわかりやすい（R9・R14・R19）
- 待ち受けにリンクされると良さそう（開くのが面倒なので）＝好意的な提案（R11）

## 3. 改善してほしい点・わかりにくかった点

**記録・操作のわかりにくさ（最多クラス）**
- どこを押せばいいか・記録の仕方に迷う／戻り方がわからない（R1・R7・R10・R22）
- 睡眠の「のこりをなぞって記録」の“のこり”が何を指すか直感的に伝わらない（R4・R17・R14）
- いつの記録なのか／何を目標にしているのかがわからない（R3）

**チュートリアル**
- 説明を読み込まないとわからない作り／操作前にもう少し説明が欲しい（R15・R19・R20・R25）
- チュートリアルが長く1回で覚えられない、「ここからまた見れるよ」の案内が欲しい（R14）
- チュートリアル3で文字と背景が重なって見づらい（R8）
- QR読み取り後すぐ始まり、チュートリアルの提示をもう少しわかりやすく（R25）

**入力の手間・情報量**
- 今日やったことの入力の手間が大きい（R10・R18）
- 文字・情報量が多く、疲れているときに見にくく継続できるか不安（R6・R23）

**細かなUI/表記**
- 「9/h」に単位がなくピンと来ない → 「9シャカ/h」等にしてほしい（R14）
- 時間割り振りに +1／−1時間ボタンが欲しい（R14）
- 睡眠だけ“なぞって指定”に違和感（R14）
- 疲れ度の入力がスッと理解できなかった（R21）
- 初期画面から既にチップが溜まっていて、追加しても変化が感じにくく「頑張りが増えた感」が薄い（R21）

**パフォーマンス**
- 動作が重くてストレス（R3）

## 4. 「限界まで頑張りすぎるのを防ぐ」目的への“足りない点”（考察に直結）

- シャカシャカする／溜めること自体が「防ぐ」に**直接は繋がっていない**気がする（R15・R16・R25）
- **認知できても「休もう」と行動を変えるところまで至らない**（R18）
- 溜まり続けるだけだと放置しそう／頑張りがたまったら何かあるのか（R19）
- 逆に、溜まるほど「やった証拠」に見えて**むしろ頑張ってしまいそう**（R7・R21）
- 限界まで溜まると、頑張りすぎを自覚して**つらくなってしまう**（R16）
- 「頑張らないといけないもの」と「頑張りたいもの」を分けたい（R11）
- 1週間のスケジュールから溜まる量を予測し、「このままだとパンパンだから今日はこのくらい休んで」「明日余裕ないから今日できることを少しだけ」等、**先回りの提案**が欲しい（R14）
- その日少しでも頑張った自分への**“褒め”フェーズ**がなく虚しい（R6）
- そのときの感情・状態を書く**プチ日記機能**で後から客観的に振り返りたい（R17）
- 円グラフで各項目にどれだけ力を使ったか見たい（R8）

## 6. 梗概で著者（小谷）が指摘していること・やりたいこと

### 考察（第7節）で述べた要点
- 当初の仮説「頑張りすぎる人は疲労に気づけない」は**支持されなかった**。多くは**気づいていても止まれない**でいた。
- 表1より、**早く気づいた群ほど回復できているわけではなく、気づきは回復に結びついていなかった**。
- したがって支援では、**気づかせること以上に、気づいた後の“回復を促す”ことが重要**。
- 頑張りすぎ経験のない回答者は体力管理や余白の確保を並行していた。目指すべきは「頑張らないこと」ではなく、**容量と回復力を保って頑張り続けられる状態**。
- よって本ツールは、**気づき＋「止める基準」の提示＋「回復の後押し」**を備えて設計する。

### プロトタイプの狙い（第8節）
- 論理的に指示するのではなく、**感覚的に「使ってみたい」と能動的に思える**ことを狙う。
- 頑張り（疲労）を行動記録から推定し、対応する絵文字を積み上げる Web アプリ。
- **画面面積＝疲労許容量（最大100）とし、8割程度で“適度”に見える**ように設計。
- 端末を振ると絵文字が揺れ、**余白があるほど気持ちよく動く**。「振れる余白を残したい」感覚で頑張る量を抑える。
- 評価は支援対象＝**回復できていない15名を中心**に整理。この層は説明「壊れず成果を出し続ける」を「心を休める」より支持（11:3）。
- 意図（行動の可視化と気づきの喚起）は伝わっていた。課題は**記録操作・チュートリアルのわかりにくさ**、**疲れているときほど情報量が多く使いにくい**点に集中。加えて**認知しても休む行動まで至らない／溜まるだけだと放置しそう**という“気づきの先の後押しの弱さ”が共通して挙がり、事前調査の知見と一致した。

### 展望（第9節）で「やりたい」と述べていること
- アンケートで挙がった**記録体験・UX を改善**する。
- 継続使用に耐える**ネイティブアプリとして再実装**する。
- **各行動の疲労度の重みづけを実感に合うよう調整**する。
- **使用前後で、休憩の頻度・活動量などの行動、および心身の状態が変化したか**を検証する。
- 結果を踏まえて改善し、**最終成果物として完成**させる。

## 7. アンケート → 展望への橋渡し（改善タスクの整理）

アンケートの声を、梗概の「気づき＋止める基準＋回復の後押し」という方針に対応づけると、改善の優先度は次のように整理できる。

**A. 記録体験・UX（“疲れているとき使いにくい”を解消）**
- 記録・操作導線をシンプルに（何を押すか・戻り方の明確化、入力の手間削減、ショートカット）
- チュートリアルの改善（対象箇所だけハイライト／短く／後から見返せる／文字と背景の重なり解消）
- 単位表記（「9シャカ/h」）、±1時間ボタン、睡眠“なぞり”の言い換え、動作の軽量化

**B. 「止める基準」の提示（“防ぐ”に直結させる）**
- 「防ぐ」に直接つながる手応えの弱さ（R15・R16・R25）に対し、**閾値・基準の可視化**を強化
- スケジュールから溜まる量を予測して**先回りで休息を提案**（R14）

**C. 「回復の後押し」（気づきの先へ）**
- 認知→休む行動への橋渡し（R18・R19）。回復行動の提案・リマインド
- 頑張りへの**“褒め”フィードバック**（R6）、感情・状態を残す**プチ日記**（R17）
- 「溜まる＝やった証拠」で逆に頑張ってしまう副作用（R7・R21）への配慮（見せ方の再設計）

**D. 枠組み・メッセージング**
- 説明は **B「壊れず成果を出し続ける」** を主軸に（支持14、対象層11:3）
- 「頑張らないといけないもの／頑張りたいもの」の区別（R11）
