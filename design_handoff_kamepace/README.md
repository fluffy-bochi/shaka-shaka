Claude Code で実装するための設計ハンドオフ資料です。この README だけで実装できるよう、画面・挙動・状態・デザイントークンをまとめています。

## 本番実装の方針（決定事項・最重要）

1. **本番は React で作る**（プロトタイプがReact構造のため移植コストが最小。Vite + React のWebアプリ／PWAを想定。将来ウィジェットが必要になったら Expo 等でネイティブ化）。
2. **UI/UXの“正”はプロトタイプ（`かめペース プロトタイプ.dc.html`）**。画面構成・遷移・見た目・挙動はプロトタイプに1:1で合わせる。`reference/index.html`（旧本番）のUIは採用しない。
3. **旧本番 `reference/index.html` から移植するのは4点だけ**：
   - Firebase ログイン（Google認証）
   - クラウド同期（Firestore への state 保存/読込）
   - Google カレンダー / ToDo 取り込み
   - シャカシャカ音（`reference/sound/syakasyaka.mp3`・衝突時に再生）
   該当コードは `reference/index.html` 内の `<script type="module">`（Firebase部）と音声再生部にある。**データ形式は旧本番と同期互換を保つこと**（既存ユーザーデータを壊さない）。
4. **疲労は日をまたいで持ち越す**：シャカの山は日付が変わっても残り、回復（休憩・睡眠・マイナス絵文字の衝突）でだけ減る。プロトタイプ・旧本番ともこの設計。
5. 進め方の推奨：本番リポジトリでブランチを切り、このバンドルを置いて Claude Code に下の「指示プロンプト例」を渡す。

## Overview（概要）

「かめペース」は、食事カロリーの代わりに **行動による“疲労”** を記録するモバイルアプリのプロトタイプです。あすけん（食事記録アプリ）の記録体験を下敷きにしています。

コアメタファーは **「疲労シャカシャカ」**：記録した行動の疲労量ぶんだけ絵文字が容器に降って積もり、休憩・睡眠などの回復でぶつかって消える。**累積疲労を数値・ゲージ・％では一切見せず**、背景に積もる絵文字の“量”だけで体感させるのが最大の特徴です。

対象ユーザー：疲れやすさをセルフケアしたい人。叱責・達成演出をしない、非評価的なトーンが設計原則。

- `かめペース プロトタイプ.dc.html` … **HTMLで作った設計リファレンス**です。そのまま本番に載せるコードではありません。見た目・挙動・情報設計の“正”として参照してください。

- `support.js` はプロトタイプ再生用のランタイムです。**本番には移植しません**（挙動確認のために `.dc.html` をブラウザで開きたいときだけ必要）。
- `CLAUDE.md` … ユーザーが繰り返し出した**恒久ルール集**。実装時の意思決定はこの表を最優先で守ってください（特に §A 情報設計、§E 絶対原則）。
- `reference/index.html` … **旧本番（バニラJS）**。Firebase・カレンダー取り込み・音の移植元。UIは参照しない。
- `reference/native_index_hybrid.html` … 旧本番にプロトタイプの記録フロー等を部分移植した中間版（参考。移植済みのバニラ実装例として検索フロー・配分バー・睡眠なぞり等のロジックが読める）。

### `.dc.html` の読み方
1ファイル内に3ブロックがあります：
- `<x-dc>…</x-dc>` … テンプレート（マークアップ）。`{{ name }}` は値の差し込み、`<sc-if>` は条件表示、`<sc-for>` は繰り返し。
- `<script data-dc-script>` 内の `class Component extends DCLogic {…}` … ロジック（Reactクラス相当）。`state`、`renderVals()`（テンプレへ渡す値/ハンドラを返す）、`componentDidMount` などがある。
- スタイルはすべて **インラインstyle**。色・寸法はそこに直接書かれています。

## 画面一覧（Screens / Views）

タブは下部固定の3つ：**シャカ / ホーム / マイページ**。記録・睡眠・登録確認・ためた回復はホーム系から遷移するサブ画面。`screens/` に各スクショあり。

### 1. ホーム（Home） — `screens/01_home.png`
- **目的**：その日の記録を時間帯ごとに俯瞰し、記録を足す入口。
- **レイアウト**：縦スクロール。背景 `#f7f4ec`。上から「睡眠カード」→「時間帯カード ×4（朝/午前/午後/夜）」。左右マージン16px、カード間12px。
- **睡眠カード**：白・角丸16px・`padding:13px 14px`。左に🛏、中央「睡眠」、右に回復量。タップで睡眠記録画面へ。
- **時間帯カード（あすけん式）**：白・角丸18px・`box-shadow:0 1px 3px rgba(27,27,24,.06)`。
  - ヘッダー行：左に42×42円アイコン（記録あり `#eaf5c9`＋右上にライム`#c4f000`のチェックバッジ／未記録 `#eef0e6`）＋時間帯絵文字、時間帯名（`font-weight:800`）、右端に**その時間帯の疲労合計**（Space Mono・オレンジ`#f5994e`・`+N`／`−N`）。
  - 記録行（`border-top:1px solid #f1efe8` 区切り）：絵文字＋タイトル、右端に**疲労量（+N/−N・オレンジ）**。※分は出さない。
  - **予定（複数タスクを内包）は📋グループ1行に集約**：`📋`＋予定名＋サブ「N件のタスク」＋右端に合計疲労。タスク個別行は出さない。
  - **予定（未確定・進行中）**はグレー表示：タイトル前に`予定`バッジ（`#a5a39a`）＋右端は `dropped/合計`（グレー）。
  - フッター：「＋記録する」（ライム文字`#7a9a00`）。タップでその時間帯の記録画面へ。
- **禁止**：累積疲労の数値・ゲージ・％・達成演出・記録率・叱責コピー。

### 2. 記録フロー（Record） — `screens/02_record_slot_pulldown.png`
記録は **時間帯 ＞ 予定 ＞ タスク** の階層。ヘッダー右上に **時間帯プルダウン**（あすけんの「間食」ピル準拠）を常時表示：絵文字＋時間帯名＋∨のピル、開くと朝/午前/午後/夜の4択（現在にチェック）。**全サブ画面で表示**し、途中でも記録先を変更可能。
入口は上から3つ（この順）：
1. **検索窓**（プレースホルダ「なにをした？（例：皿洗い、会議）」）
2. **予定から**（大カテゴリと同じ**縦リスト**。📋＋予定名＋「N件・合計時間」＋›。末尾「＋予定をつくる」）
3. **大カテゴリから**（縦リスト。単色Materialアイコン＋名前＋サブ。末尾「＋大カテゴリを追加」）

- **大カテゴリ→小カテゴリ**：カテゴリを選ぶと小カテゴリ（やったこと）一覧。行タップで選択（＋→✓）。一部は程度ポップアップ（中央カード）。
- 大カテゴリ例：いどう / しごと・学業 / 家事 / 対人・イベント / **休憩・回復**（お茶・入浴・昼寝/仮眠・ストレッチ/ヨガ・深呼吸/瞑想・ぼーっとする＝fh<0の回復系）。
- **大カテゴリもユーザーが増やせる**（追加導線必須）。小カテゴリ新規はコピー＋「くらべてどう？」3択、空欄新規は不可。

### 3. 検索（Search） — `screens/04_search_intensity.png`
- **検索窓タップ＝最初から検索窓を10個並べる**（複数キーワード一括検索）。入力すると次行が有効化。
- 結果は**キーワードごとに上から表示**。候補の＋で採用すると**そのキーワード群は消えて下が繰り上がる**。
- 候補5件超は「もっと見る」。
- **候補が無い時は2択**：「新しく登録（次回から候補に出る）」／「だいたいで登録（きまった目安の疲労）」。
- 選び終えると **登録を確認** 画面へ合流。

### 4. 登録を確認（Confirm / 記録の確定UIは1つに統一） — `screens/03_confirm_timemode.png`
**予定を選ぶ／カテゴリでタスクを選ぶ／検索で選ぶ → すべてこの画面に合流**する。
- 採用アイテムを縦リスト表示（×削除・**強度チップ**）。「＋メニューを追加」。
- **時間の割り振り**カード：
  - 上部に **「所要時間／時刻」切替**（セグメント）。
  - *所要時間モード*：合計分の −10/−1/＋1/+10 ステッパー（**1分単位**）。
  - *時刻モード*：開始→終了の `type=time` 入力。各アイテムは時間帯に順に割り当て。
  - **配分バー**：色分けセグメント＋**つまみを左右ドラッグで1分単位調整**（等分固定・ステッパーだけに戻さない）。下に各行の分・疲労。
- **強度チップ**（＝あすけんの分量入力に相当）：行動ごとに設問が変わる（例 通勤＝混み具合／座れた？、仕事＝長さ／気疲れ、家事＝量、運動＝強度）。選択で疲労量が係数調整される。
- 確定ボタンは状況で「きろくする／予定を追加」に切替。

### 5. シャカシャカ（Shaka・物理演算）
- **記録確定→シャカ画面へ遷移し、上から絵文字が降って積もる**。matter.js の重力落下＋衝突。
- **既に積もっている絵文字はそのまま静止**、**新しく記録した分だけ上から降らせる**（全部を落とし直さない）。
- 絵文字1個＝疲労1。回復（fh<0）・睡眠・「後から回復(after)」は積もった絵文字を末尾から差し引く。
- 背景の「固定／動かす」は**マイページ**で切替（固定＝落下が落ち着いたら演算停止で静止／動かす＝演算し続ける）。ボカシスライダーは無し。

### 6. 睡眠記録（Sleep）
- 背景はホームと同じ `#f7f4ec`。上部の説明文は出さない。
- 縦の残量バーを**画面をなぞって**設定。右側の具体数値（残N・回復−N）は**出さない**（絵文字量とラインだけ）。
- CTA は「**これくらい回復した**」。回復量＝就寝前 − いまの残量（残量から逆算）。時刻は日付を閉じる用途のみ。

### 7. ためた回復（Collected）
- 回復で消した絵文字が縦に長いスクロール世界に積もる（matter.js）。上端に合計。

### 8. マイページ（My Page）
- プロフィール、きろく設定、つかれの計算、**ひょうじ：背景のシャカ「固定／動かす」トグル**、ログアウト。

## Interactions & Behavior（挙動）

- **画面遷移**：`state.screen`（`home`/`record`/`confirm`(旧)/`shaka`/`sleep`/`collect`/`mypage`）で切替。下タブは常設。
- **記録の合流**：カテゴリ`goConfirm`・予定`openPlan→planToConfirm`・検索`goSearchConfirm` → いずれも `searchStep:'confirm'`（＝統一確認UI）。`confirmOrigin`（'search'/'cat'/'plan'）で戻る先を分岐。
- **配分バードラッグ**：`onFracDrag(k)` が隣接2セグメントの比率を1分刻みで再配分（最小1分保証）。
- **予定の時間進行（index.html と統一）**：時刻モードで終了が未来なら `planned:true, dropped:0` で記録。1秒ごとの `advancePlans()` が経過に応じて `dropped` を増やし（`planUnitsDue`＝from→toを疲労数で等分）、シャカ表示中なら降ってくる分だけ `addFallingBodies` で落下。`dropped>=|fat|` で `planned:false`（記録に昇格）。ホームはグレー`予定`バッジ＋`dropped/合計`。
- **物理の固定/動かす**：`homeMotion` false＝落下後 `freezeMotion()` で Runner 停止（静止）、true＝`resumeMotion()` で走らせ続ける。
- **アニメ**：トーストは `pop .25s ease`。落下は matter.js（重力 y=1、restitution 0.35、frictionAir 0.01）。

## State Management（状態）

`Component.state` の主なもの：
- `screen` … 現在画面。
- `slots` … 時間帯×4。各 `{ id, emoji, name, records:[…] }`。`record` = `{ name, glyph, min, fat, plan?, after?, planned?, dropped?, from?, to?, _new? }`。
- `slotId` / `slotMenuOpen` … 記録先の時間帯／ヘッダープルダウン開閉。
- `catId` / `cart` … 選択中カテゴリ／小カテゴリの選択（程度含む）。
- `customCats` / `customPlans` / `customActions` … ユーザー追加のカテゴリ・予定・検索候補。
- `searchStep`（'input'/'results'/'more'/'confirm'）/ `keywords`(最大10) / `searchCart` / `resolvedIdx` / `moreKw` … 検索フロー。
- `searchTotalMin` / `searchFracs` / `confirmMode`('duration'/'time') / `startTime` / `endTime` … 割り振り。
- `intensityId` … 強度ポップアップ対象。
- `collected` / `sleepTrim` / `residual` … 回復プール・睡眠回復・睡眠残量。
- `homeMotion` … 背景シャカの固定/動かす。
- `dayOffset` … 過去日表示（プロトタイプ内デモ）。

**データ計算のキモ**：
- `effFh(item)` … 強度チップの係数を掛けた1時間あたり疲労。`fat = effFh × (分/60)`。
- `pileGlyphs()/pileGlyphsMarked()` … 記録から積む絵文字配列を生成（planned は dropped 分だけ、回復・afterで末尾から差し引き）。
- `groupRecords()` … ホーム表示用に予定を📋グループ化＋planned進捗を整形。

## §絶対原則（必ず守る・CLAUDE.md 由来）

- 記録率・網羅率・連続記録・達成演出を**出さない**。
- 未記録・疲労超過に警告色や叱責コピーを**使わない**（100超もそのまま）。
- 疲労量の数値を**直接入力させない**（数値入力は時間だけ）。
- 累積疲労の数値・ゲージ・％を**出さない**（背景シャカの絵文字量だけ）。

**本番のみでよい（プロトタイプに作らない）機能**：ログイン/クラウド同期、Googleカレンダー/ToDo取り込み、衝突音。「そろえる」道具は不要。
