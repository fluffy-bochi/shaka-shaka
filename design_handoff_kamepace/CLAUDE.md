# かめペース — 恒久ルール（毎ターン必ず参照）

ユーザーが繰り返し指示したのに反映漏れがあり苛立たせた。作業前に必ずこの表を読み、該当項目を守る。詳細な作業流儀は `skills/かめペース作業スキル_for_opus.md`。

## A. 記録の情報設計（最重要・何度も指示済み）
- **記録の階層は 時間帯 ＞ 予定 ＞ タスク**。例：朝 ＞ 通勤（＝予定のタイトル。テンプレとして保存できる）＋所要時間 ＞ タスク（徒歩・電車・バス）
- 予定＝テンプレ化できるまとまり。タスク＝その中の個別行動
- 記録UIの選択順：記録ボタン → 大カテゴリ → 小カテゴリ（やったこと）→ ものによって程度ポップアップ（中央カード型・あすけん分量風）
- **記録の入口は「大カテゴリから／予定／検索」の3つ**（順序もこの通り。予定は大カテゴリと検索の間）
- **記録画面のヘッダー右上に時間帯（朝／午前／午後／夜）を切り替えるプルダウン**（あすけんの「間食」ピル準拠）。絵文字＋時間帯名＋∨のピル型、開くと4択（現在の時間帯にチェック）。記録の全サブ画面で常時表示し、途中でも記録先を変更できる
- **「予定」入口＝カレンダー式の予定を追加/選択できる**。予定は複数の行動（タスク）を内包する（例：通勤 ＞ 徒歩・バス・電車）。予定を選ぶと内包タスクをまとめて記録できる
- **予定の入口は大カテゴリと同じ縦リストで並べる**（横スクロールのチップにしない）。末尾に「＋予定をつくる」
- **予定を選ぶ／大カテゴリでタスクを選ぶ → どちらも検索の「登録を確認」画面（時間割り振りバー付き）に合流**する（記録の確定UIは1つに統一）
- **Googleカレンダー同期もこの予定フォーマットで取り込む**（カレンダーの予定＝かめペースの予定として、内包タスク付きで扱う）
- **大カテゴリもユーザーが増やせる**（追加導線を必ず用意）
- 小カテゴリ（タスク）の新規はコピー＋「くらべてどう？」3択。空欄新規は不可
- **タスク・予定の時間は1分単位で記録**（0.5h刻みにしない。分入力 or 分単位ステッパー）
- **検索窓をタップ＝最初から10件の検索窓を並べる**（複数キーワードを一括検索）。結果はキーワードごとに上から表示し、1つ選ぶとそのグループは消えて下が繰り上がる
- **検索で候補が無い時は2択**：「新しく登録（次回から候補に出る）」か「だいたいで登録（きまった目安の疲労）」
- **時間の割り振りは1分単位＋バーのつまみを左右ドラッグで直感調整**（等分固定・ステッパーだけの方式に戻さない）
- **休憩・回復カテゴリには回復系タスクを揃える**（お茶・入浴・昼寝/仮眠・ストレッチ/ヨガ・深呼吸/瞑想・ぼーっとする 等、fh<0）

## B. ホーム画面
- 予定・タスクの記録は**縦に並べる**（あすけんホーム参照。チップ横並び・折返しにしない）
- カードは**あすけん式**：左に円アイコン（記録ありは薄ライム#eaf5c9＋右上ライムのチェックバッジ、未記録は#eef0e6）＋時間帯名（900）＋右にsumText、区切り線、行（絵文字＋タイトル＋右端に疲労）、フッターに「＋記録する」
- **記録行の右端は「分」ではなく疲労量（+N／−N・オレンジ）を表示**（分は出さない）
- **予定が複数タスクを含む場合は予定だけ表示**（📋＋予定名＋「N件のタスク」＋合計疲労。タスク詳細は出さない）
- ホーム背景＝シャカ画面＝睡眠記録画面＝#f7f4ec（screenBgで統一）
- 表示は 時間帯 ＞ 予定（📋グループ）＞ タスク の入れ子。予定の下にタスクをぶら下げる
- 累積疲労の数値・ゲージ・%は出さない（背景シャカの絵文字量だけ）
- **背景シャカの「固定／動かす」切替はマイページに置く**（ボカシ量スライダーは不要）。固定＝落下が落ち着いたら演算停止で静止、動かす＝演算し続ける

## C. 記録後の遷移
- **記録を確定したらシャカシャカ画面に移り、上から絵文字が降ってくる**演出を出す（記録した分が降って積もる）
- **既に積もっている絵文字はそのまま（下に静止配置）、新しく記録した分だけ上から降らせる**（全部を落とし直さない）
- **所要時間の設定は「所要時間／時刻」を切替できる**。時刻モードで終了が先の時刻なら「予定」として扱い、index.html と同じ挙動に統一する：予定は時間どおりに絵文字を1個ずつ降らせ、終了時刻で記録に昇格。予定はホームでグレーの「予定」バッジ＋dropped/合計で表示
- **本番のみでよい機能（プロトタイプに作らない）**：ログイン/クラウド同期、Googleカレンダー/ToDo取り込み、衝突音。「そろえる」道具は不要

## D. 睡眠記録画面
- 上部の説明文「おはよう。いま、どれくらい残ってる？画面をなぞって〜」は**消す**
- CTA「この残量できょうをはじめる」→「**これくらい回復した**」に変更
- 背景は**ホーム画面の背景と同じ**（#f7f4ec）にする
- 縦の残量バー右側の具体表記「残19・回復−46」などの**数値は消す**（残量は絵文字の量とラインだけで表現）
- 回復量＝就寝前 − いまの残量（残量から逆算）。時刻は日付を閉じる用途のみ

## E. 絶対原則（§1・全画面共通）
- 記録率・網羅率・連続記録・達成演出を出さない
- 未記録・疲労超過に警告色や叱責コピーを使わない（100超もそのまま）
- 疲労量の数値を直接入力させない（数値入力は時間だけ）
- 累積疲労の数値・ゲージ・%を出さない

## F. デザイン言語
- フォント：Zen Kaku Gothic New（本文）／Space Mono（数値・ラベル）／Archivo（大数字）
- 色：ライム #c4f000（アクション・体）、ピンク #ff5fa2（心）、背景 #f7f4ec、墨 #1b1b18、白カード
- リスト行：裸のMaterial Symbols（FILL 1・カテゴリ色）＋名前＋Space Monoサブテキスト＋境界線 #f1efe8。選択中＝#fbfdf0＋左端ライム
- アイコンはZaim風単色（Material Symbols Rounded）。絵文字は枠アイコン🌅☀️🌤🌙🛏と装飾のみ
- 絵文字サイズ統一（画面全体に約100個収まる基準）。絵文字1個＝疲労1

## G. ファイル
- `かめペース UI v1（…）.dc.html`：静的モック一式。新ターンのセクションは最上部に追加（turn番号＋画面ID、idバッジ＋アンカー）
- `かめペース プロトタイプ.dc.html`：触れるプロトタイプ（下部タブ常設・ホーム中央・matter.js物理シャカ）。**設計の正＝ベース**。音・同期は入れない（デザイン検証用に常にクリーンに保つ）
- `native/index.html`：**本番用ビルド（バニラ）**。プロトタイプのUX（時間帯スロット＋ヘッダーの時間帯プルダウン＝朝/午前/午後/夜）をベースにしつつ、index.html由来の Firebase同期・Googleカレンダー/ToDo取り込み・シャカシャカ音(syakasyaka.mp3) を採用。
  - モデル橋渡し：内部データは index.html の `entries`(from→to)/`tasks` のまま（同期・音・取り込みは無改造で維持）。時間帯スロットは entries の from時刻でバケツ分け（朝6-9/午前9-12/午後12-18/夜18-）した「見え方」。通常記録＝選んだ時間帯の基準時刻に配置、時刻モード/予定＝実時刻(from→to)で planned 進行。
  - 音・同期・取り込みは**本番(native)のみ**。プロトタイプ(dc)には入れない。

## H. 実装のハマりどころ
- メソッド書き換え時、シグネチャ行（`foo() {`）をold/new両方に入れると重複行ができDCの logic eval が壊れる（画面が真っ白＝navのみ）。中身だけ置換する
- logic破損は get_webview_logs の "logic class eval FAILED" で判る。構文疑いはASCII名にコピー→`new Function(body)`で確認
- 物理ループ中の画面は eval_js の多段awaitが10秒タイムアウトしがち
