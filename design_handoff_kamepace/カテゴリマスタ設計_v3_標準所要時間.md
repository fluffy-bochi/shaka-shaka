# カテゴリマスタ設計 v3 — 標準所要時間と検算の見直し

v2からの追加・変更：行動ごとの標準所要時間（defaultMin）／「長さ・量」系の強度チップ廃止（時間との二重カウント解消）／ペルソナ3人の再検算。

---

## 1. 問題意識

- 現実装はカテゴリ経由の初期値が**一律60分**（回復系と買い物のみ30分）。「皿洗い60分」のような非現実的な初期値になっており、ユーザーが毎回時間を直す摩擦が生まれている。
- 強度チップに「長さ（短め/ふつう/長め）」「量（すこし/ふつう/たくさん）」があるが、これは**時間と同じものを二重に聞いている**。長い会議は時間を長くすれば表現できるのに、係数でも増やせてしまい、疲労が二重に膨らむ。
- 検索経由の初期値も一律30分×件数で、行動の実態と無関係。

## 2. 方針

1. **各行動がマスタに標準所要時間 `defaultMin` を持つ。**
   - 「その行動を1回やるときの、だいたいの中央値」を5分刻みで設定する。
   - 目的は記録の摩擦を減らすこと。正確さは求めない（あすけんの「だいたいでOK、あとから変えられます」の思想）。時間の調整手段（±ステッパー・キーボード入力・配分バー・時刻モード）は現状のまま。
2. **初期値の適用箇所**：カテゴリ経由（goConfirm）・検索経由（commitSearch の初期合計）・予定作成時のタスク時間。複数選択時の合計 = 各 defaultMin の和。
3. **強度チップから「時間の言い換え」を排除する。**
   - 廃止：仕事系の「長さ」、家事の「量」、皿洗いの程度ポップアップ（すくなめ/多め）→ すべて時間で表現する。
   - 残す：時間と独立な**密度・心理系**のみ。
     - 通勤・移動：混み具合／座れた？
     - 仕事・バイト：気疲れ（少ない/ふつう/多い）
     - 育児：機嫌
     - 運動：強度（軽め/ふつう/ハード）
   - 検索バリエーション（電車・座れた/立ち 等）は密度差なので残す。
4. **コピー式「くらべてどう？」は /h の密度比較であることを明示する。**
   - 質問文に「**同じ時間やったとして**」を添える（例：「講義とくらべて、同じ時間やったとして、こころは？」）。
   - defaultMin もコピー元から引き継ぎ、作成時に変更できるようにする（任意・後回し可）。
5. **1回分の目安表示**：一覧の「目安 +9/h」は分かりにくいので「**目安 +5（30分）**」の形式に変更する（rate × defaultMin を四捨五入・最低±1）。

## 5. 比較まわりの整理（何がどこで効くか）

| 変えたい体感 | 使う手段 |
|---|---|
| 長くやった／短くやった | **時間**（ステッパー・キーボード・配分バー・時刻） |
| 同じ時間でも密度が濃い | **強度チップ**（混雑・気疲れ・機嫌・運動強度のみ） |
| その行動が自分にとって恒常的に重い/軽い | **すききらい**（×1.3/×0.7）・**コピー式で自分専用の行動を作る** |
| 体質として疲れやすい/回復しやすい | **個人係数**（体・心×疲労/回復の4係数） |

- 「長さ」「量」チップと皿洗いの程度ポップアップは廃止（時間へ一本化）。
- コピー式の質問文は「同じ時間やったとして」を明記。

## 6. 実装タスク

1. マスタの各行動に `defaultMin` を追加（§3の表）。
2. `goConfirm`：durOf の一律60/30 → `defaultMin` を使用。
3. 検索経由：`selectSearchItem`/`commitSearch` の初期合計 30×件数 → 選択行動の `defaultMin` 合計（検索候補にも defaultMin を持たせる）。
4. 行動一覧のサブ表示を「目安 +N/h」→「目安 +N（M分）」へ。
5. 強度チップ：仕事系「長さ」・家事「量」を削除（気疲れ等は残す）。皿洗いの程度ポップアップ（degree）を削除。
6. コピー式の質問文に「同じ時間やったとして」を追記。作成した行動の defaultMin はコピー元を引き継ぐ。
7. ヘルプのペルソナ3人を§4の数値に更新。
8. ゲストサンプル（大学生の1日）をユウタ§4の構成に合わせて更新。
