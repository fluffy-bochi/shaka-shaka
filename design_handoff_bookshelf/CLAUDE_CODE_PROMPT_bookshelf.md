# Claude Code 実装プロンプト — 本棚タイムライン

以下をそのまま Claude Code に渡す。

---

かめペース（疲労を絵文字でためる記録アプリ）の本番ビルド `native/index.html` に、新しいタイムライン画面「がんばりの本棚」を実装してください。

## まず読むもの（この順で）
1. `design_handoff_bookshelf/指示書.md` — 仕様の正。全部読む
2. `design_handoff_bookshelf/reference/本棚タイムライン 案.dc.html` — 動くデザインモック。ブラウザで開き、**#1a（縦の本棚）と #3a（横の採用版）だけ**を再現対象とする（2a/2b/1b は検討過程、実装しない）。スタイル値・配置・挙動はこのモックの inline style を正として写す
3. `native/index.html` — 既存の本番コード。`entries`(from→to)/`tasks` のデータモデル、Firebase 同期、時間帯バケツ（朝6-9/午前9-12/午後12-18/夜18-）を把握する

## やること
- 下部タブに「本棚」を追加し（auto_stories アイコン）、本棚画面を実装する
- 縦持ち = 1a のレイアウト、横持ち = 3a のレイアウト。**デバイスの orientation で自動切替**（モックの回転ボタンは実装しない）
- データは既存 `entries`/`tasks` から導出（指示書 §2）。乱数モックデータは移植しない
- fav（付箋）と diary（日記）を dateKey ベースで localStorage に永続化（既存の保存の仕組みがあるならそこに載せる）
- 機能一覧: 日別/月別切替・今日ボタン・月セレクト・ソート（日付/多い/少ない）・あふれた日フィルタ（11個以上）・付箋の日フィルタ・本タップ→その日シート（縦は下からポップアップ、タブバーは覆わない）・時間帯タイムライン・円グラフ・日記・月別の疲労/回復TOP5・週番号（月曜始まり・ゴールド）

## 守ること（絶対）
- 累積疲労の数値・ゲージ・%を出さない。警告色・叱責コピーなし。達成演出なし
- 既存の Firebase 同期・Googleカレンダー/ToDo取り込み・シャカ音のコードは**無改造**で維持する
- 既存画面のスタイル・挙動を壊さない（本棚は追加のみ）
- フォント/色は指示書 §6 のトークンに従う（新しい色を発明しない）

## 完了条件
- 縦・横どちらの向きでもモックと見比べて同じ構成・同じ密度で表示される
- 実データ（記録がある日）で絵文字が下から積もり、予定はグレー、今日にライム枠が付く
- fav・diary がリロード後も残る
- 記録ゼロの日は空の透明な本として表示される（エラーにならない）
