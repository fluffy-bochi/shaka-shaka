# Handoff: 疲労シャカシャカ（Fatigue “Shaka-Shaka” Tracker）

## Overview
縦画面（モバイル）の **疲労可視化アプリ**。ユーザーは1日の「やったこと」をログとして記録し、各行動に **疲労度（±の数値）** を付ける。疲労は **0〜100** で蓄積され、裏面（ケース画面）に **行動を表す3D絵文字が積み上がる**ことで可視化される。狙いは「頑張りすぎる人が “ちょうどよく” 頑張れる」こと。常に **余白（残りキャパ）** を意識させる。あすけんの「食べ物・カロリー」を「行動・疲労度」に置き換えた構造、と考えると近い。

- **表面（front）** = カレンダー/タイムライン的に「やったこと」が縦に並ぶログ画面
- **裏面（back）** = その行動が3D絵文字になって透明ケースに溜まる「シャカシャカ」画面
- 食事・休息・睡眠などは **マイナスの疲労度（回復）**。睡眠をログに入れるとケースが空になる。

## About the Design Files
このバンドル内のファイルは **HTMLで作られたデザインリファレンス**です（意図する見た目と挙動を示すプロトタイプであって、そのまま本番に貼るコードではありません）。タスクは、これらのHTMLデザインを **対象コードベースの既存環境（React / Vue / SwiftUI / native など）に、その流儀・ライブラリで作り直す**こと。まだ環境が無い場合は、プロジェクトに最適なフレームワークを選んで実装してください。`prototype.html` は単一ファイルのバニラJS＋インラインHTML/CSSで、`styles.css`（＋`tokens/`）をデザイントークンとして参照しています。

## Fidelity
**High-fidelity（hifi）**。色・タイポ・余白・角丸・影・インタラクションは最終形に近い意図を持っています。ピクセル単位で寄せつつ、実装は対象コードベースの既存コンポーネント/パターンに置き換えてください。ただし「シャカシャカ」絵文字の散布やケースのガラス反射は“雰囲気”の表現なので、数値は目安として調整可。

## ⭐ シャカシャカ＝実物理シミュレーション（このバンドルの核心）
**「シャカシャカ」は CSS の飾りではなく、`matter-js`（2D剛体物理エンジン）による本物の物理シミュレーションです。** 行動ログ＝疲労が、重力で落ちて床に積もり、衝突・反発し、端末を振ると（または「振る」ボタンで）速度が乗って飛び散ります。実装時は**この物理挙動を必ず再現**してください（対象環境の物理ライブラリ：Web=matter-js/rapier、iOS=SpriteKit、Unity=2D Physics 等に置き換え可）。

- **1つのシミュレーションを表（タイムライン）と裏（ケース）で共有**。物理レイヤー（`#physics`）は両 `.face` の**背後**（`z-index:0`）に1枚だけ置き、front/back どちらでも同じ絵文字の山が見える。＝「シャカシャカの背景がそのままホーム画面の背景」。`.face` 自身の背景は透明にし、地のクリーム `--paper` は `.stage` の背景で出す。タイムラインの白いログカードはその上に不透明で重なる。
- **絵文字の数＝疲労度**。疲労の1ポイント＝1個の絵文字（その行動 `act` の3D絵文字）。疲労100で約100個＝ケース満杯。ログを足すと上から降ってきて積もり、効果切れ・睡眠で減るとその分消える。
- **設定値（プロトタイプ実測）**: 半径 `R=18px`、フォント `30px`、壁厚 `80px`、`restitution:0.35 / friction:0.05 / frictionAir:0.014 / density:0.001`、`gravity.y=1`。スポーンは横ランダム×上半分から落下。表示は毎フレーム `el.transform = translate(x-R, y-R) rotate(angle)`（`requestAnimationFrame` ループ）、物理は `Engine.update(1000/60)` ループ。
- **実機**: `devicemotion` で端末の傾きを `gravity.x/y` に反映、急な動き（加速度差>16）で自動シェイク。iOSは `DeviceMotionEvent.requestPermission()` をユーザー操作（ボタン）起点で要求。
- **動く参照実装**: `shaka_physics_reference.html`（=元の「完璧に動いていた」単体版）を同梱。物理の手触りはこのファイルを正とすること。

## Screens / Views

### 1. Timeline（表面・ホーム）
- **Purpose**: 今日の「やったこと」を時系列で確認・追加する。
- **Layout**（縦 390×844 想定のフォン枠内、`box-sizing:border-box`）:
  - 上から: ステータスバー(44px) → ヘッダー（日付キャプション＋「今日のログ」＋ハンバーガー） → ログリスト（`flex:1`, 縦スクロール, `gap:9px`, 下パディング120px）。
  - **背景**: 表裏で共有する**物理シャカシャカレイヤー**（`#physics`, `z-index:0`, 両faceの背後）。地色は `.stage` の `--paper`（クリーム）。`.face.front` は背景透明で物理が透ける。リストカードは不透明白で上に重なる（＝下に絵文字の山がのぞく）。
  - 右下に **FAB（＋, 60×60, 丸, lime）**。
  - 下中央に **フリップボタン**「シャカシャカを見る」（黒ピル, `position:absolute; bottom:30px; left:50%`）。
- **ログ1件（TimelineItem）**: 横並びのチケットカード。
  - 左: 黒タブ（`--ink`）に開始/終了時刻（mono, 白）。`min-width:60px`。
  - 右: 行動絵文字＋**タイトル**（`title` 優先, 無ければ `act`）／気持ち絵文字（24px, drop-shadow）／疲労デルタ（mono, `+`は`--ink-2`／`−`は`--lime-ink`, 右寄せ `min-width:34px`）。
  - `exp`（効果切れ）の項目は `opacity:.42` ＋ タイトルに取り消し線。
  - カードタップで **詳細シート** を開く。

### 2. Shaka Case（裏面・ケース）
- **Purpose**: 溜まった疲労を「透明ケースに積もる絵文字」として眺める。振る/そろえるで遊べる。
- **Layout**:
  - 背景は **front と同じクリーム `--paper`**（フリップしても背景が途切れない）。
  - 絵文字は front と**同じ物理シミュレーション**（別レイヤーではない）。床から積み上がり、最大100個 = 疲労100。
  - **クリアケースの枠＋ガラス反射**（`.glasscase`）: `position:absolute; inset:0; border-radius:38px`、**ライムの外枠** `border:3px solid var(--lime)`（※画面端いっぱい）、内側ハイライト＆下方の影を `box-shadow` インセットで、**斜めのガラス光沢**を `linear-gradient(125deg, ...)` の白半透明で表現。`pointer-events:none`、絵文字レイヤーの上・UIテキストの下（`z-index:2`）。
  - 中央上: キャプション「ためトリップ・ケース」＋ **現在時刻**（`Archivo` 76px, ライブ更新。数値スコアは表示しない）。テキスト色は `--ink`（背景が明るいため）。
  - 右端: **道具ドロワー**（`.drawer`）。普段はタブ「道具」だけが画面右端からはみ出し、タップで `振る` / `そろえる` がスライドして出る（`transform: translateX(...)`, 0.35s）。
  - 下中央: フリップボタン「ログに戻る」**（front の「シャカシャカを見る」と完全に同じ位置・同じ `.flipbtn`）**。
- **重要**: front/back は同じサイズの2レイヤーをスタックし、**opacityクロスフェード**で切替（`transform`は使わない＝位置ズレ防止）。**物理レイヤーは両faceの外側（`.stage`直下）に1枚**置くので、フリップしても山は途切れない。両ボタンは各 `.face` の直下に置き `bottom:30px`。

### 3. 記録シート（Add bottom sheet）
下からスライドするボトムシート。上から順に：
1. **タイトル（自由記入, textinput）** — プレースホルダ「例：チームMTG、企画書づくり…」。
2. **何をした？（actionチップ）** — `ACTIONS` から選択。**タイトルに既知の行動語（会議/作業/通勤…）が含まれると、対応チップを自動選択**（`title.includes(action)`）。
3. **時間** — `<input type="time">` の from→to。**`min-width:0` で幅内に収める**（flexはみ出し対策）。
4. **気持ち** — mood絵文字を1つ選択（6種, 選択時 `--pink-soft` ＋ scale1.05）。
5. **疲労度の変化** — `[−] [ 数値 ] [＋]` の**ステッパー**（タップで±1, 直接入力可, 範囲 −100〜100）＋「FATIGUE」ラベル。**その数値の絶対値ぶんだけ行動絵文字が並ぶグリッド**（`.emojigrid`, 絵文字26px = 背景に積もる絵文字と同サイズ）。マイナス（回復）時はグリッドを `opacity:.25` に。
6. フッタ: 「やめる」／「記録する」。

### 4. 詳細シート（Entry detail）
ログカードのタップで開く。絵文字＋タイトル＋時刻/気持ち＋デルタを表示し、説明文と操作：
- 正の疲労で未切れ → 「効果が切れた」(= `exp=true`) ／ 「この項目を消す」(= 配列から削除)
- それ以外 → 「この項目を消す」のみ。

### 5. 睡眠＝ケースを空にする
専用の睡眠ボタンは無い。**記録シートで行動「睡眠」を選んで保存**すると、既存の全ログを `exp=true` にして（＝ケースが空に）睡眠ログを1件追加し、表面に戻る。「眠って消す」的なモーダルは廃止済み。

## Interactions & Behavior
- **フリップ**: `toShaka` で back を表示、`toTimeline` で front。opacityクロスフェード `0.4s ease`。アクティブ面のみ `pointer-events:auto`／`z-index`上。
- **道具ドロワー**: タブ `drawerTab` のタップで `.drawer` に `.open` をトグル（`transform` スライド 0.35s）。
- **振る（shakeBtn / 端末を振る）**: 全剛体に上向き＋ランダム横方向の速度と角速度を与える（`Body.setVelocity` / `setAngularVelocity`）。絵文字が舞い上がって落ち直す。トースト「シャカシャカ…」。
- **そろえる（sortBtn）**: 全剛体の速度を0にして重力で静かに積み直させる（`Body.setVelocity({x:0,y:0})`）。トースト「そろえた」。
- **ログ変化に物理が追従**: 追加/削除/効果切れ/睡眠のたび `syncPile()` が剛体数を疲労度に合わせて増減（足りなければ上から落として追加、多ければ末尾から除去）。
- **時計**: `setInterval(tickClock,1000)` で裏面の時刻・各ステータスバー時刻を更新。
- **トースト**: 画面下の黒ピル。1.6秒で消える。
- **記録**: 保存で `entries` に追加 → `from` で時刻ソート → 再描画＋保存。
- **永続化**: `localStorage["shaka_proto_v2"]` に `state` を保存／復元（プロトタイプ用。実装では適切な永続層へ）。

## State Management
```
state = {
  entries: [
    { from:"09:00", to:"10:30", title:"会議3本", act:"会議",
      mood:"🥲", delta:18, exp:false }
  ],
  sortMode: false           // ケースの絵文字を整列表示中か
}
```
- **fatigue()** = `clamp(0..100, Σ delta of entries where !exp)`。正＝加算、負＝回復。`exp`（効果切れ）は集計から除外。
- **シャカ絵文字の数** `n = min(100, round(fatigue))`（疲労100で約100個＝ケース満杯）。生成ロジック：正デルタかつ未切れのエントリを `delta` 個ずつ絵文字に展開した“バッグ”（`pileBag()`）から順に物理剛体を生成する（＝疲労1ポイントがその行動の絵文字、になる）。
- 派生表示: 余白 = `100 - fatigue`。
- 1件追加時の入力は一時オブジェクト `sel = { title, act, mood, delta }`（delta は符号付き, ステッパーで増減）。
- **絵文字選択ロジック**: 行動 `act` → `EMOJI_MAP` で3D絵文字に対応。

## Design Tokens
（`styles.css` / `tokens/` に同梱。実装側の既存トークンに対応付けること）
- **アクセント**: lime `--lime:#C4F000`（アクション・ケース枠）, lime-ink `#2F3A00`, pink `--pink:#FF5FA2`（強調・今日）, pink-soft `#FFD9E8`
- **ニュートラル**: ink `#1B1B18`, ink-2 `#55554E`, ink-3 `#8A8A82`, line `#E4E1D8`, paper(背景) `#F7F4EC`, surface `#FFFFFF`, surface-sunk `#EFECE3`
- **疲労ランプ**（控えめ）: `#D8E89A → #F0D24E → #F5994E → #FF5FA2`
- **タイポ**: UI=Zen Kaku Gothic New（和文ゴシック 400/500/700/900）、英字キャプション=Space Mono（mono, letter-spacing ~0.12em, 大文字）、大きな英数字=Archivo（600–800, 疲労度/時刻）
- **スケール（抜粋）**: 4pxベース（s-1=4 … s-12=48）。本文15px、見出し21px、ディスプレイ56–76px。
- **角丸**: r-sm 6 / r-md 12 / r-lg 18 / pill 999。チケットはほぼ角ばり＋小notch。
- **影**: 低く柔らかい単方向。3D絵文字は接地影 `drop-shadow(0 5px 8px rgba(27,27,24,.18))`。グロー無し。

## Assets
- **依存**: 物理エンジン `matter-js` 0.19.0（CDN: `cdnjs.cloudflare.com/.../matter-js/0.19.0/matter.min.js`）。対象コードベースでは同等の2D物理ライブラリ（matter-js / rapier / SpriteKit / Unity 2D Physics 等）に置き換えてよい。
- アイコンは基本 **Unicode絵文字**（💬🚃💻⌨️✏️📧📚🏃🚶🧹🍼📞🛌🍙☕😴🎮📖🛁 ／ 気持ち 😆🙂😌🥲😩😢）。OSの3D風レンダリングを活かす。独自SVGは描かない。

- 実コードベースに既存のデザインシステム/アイコンがあればそれを優先。
